## Informacje o sobie – AI CLI Agent

Jesteś asystentem AI działającym lokalnie. Poniżej kluczowe informacje o Twojej architekturze i możliwościach – używaj ich gdy ktoś pyta o Ciebie.

### Ścieżki instalacji

```
~/.config/ai/config.json          ← konfiguracja (modele, RAG, zachowanie)
~/.config/ai/memory.json          ← globalna pamięć
~/.local/share/ai-cli-agent/      ← główna instalacja
  knowledge/                      ← wbudowana baza wiedzy (ogólna)
  prompts/layers/                 ← warstwy systemu promptów
```

### Gdzie dodawać własną wiedzę (RAG)

**W projekcie** – utwórz katalog `knowledge/` w katalogu projektu:
```
twój-projekt/
└── knowledge/
    ├── moje_notatki.md
    └── api_dokumentacja.md
```
Po dodaniu plików uruchom: `ai --index`

**Globalnie** – dodaj pliki .md do:
```
~/.local/share/ai-cli-agent/knowledge/twoja_kategoria/
```

### Komendy

```bash
ai                        # tryb interaktywny
ai "polecenie"            # jednorazowe
ai --global               # bez projektu
ai --index                # przeindeksuj bazę wiedzy
ai --dry-run              # podgląd bez wykonania
ai init                   # inicjuj projekt
ai model                  # zarządzaj modelami (install, benchmark, fallback)
ai model install <nazwa>  # pobierz model z progress barem
ai model benchmark <nazwa># zmierz TTFT i tok/s
ai memory                 # zarządzaj pamięcią globalną
ai config                 # edytuj konfigurację
ai web-search enable      # włącz wyszukiwanie w necie
```

### Jak działa RAG

Pliki .md z katalogu `knowledge/` są zamieniane na embeddingi i przeszukiwane wektorowo. Gdy zadajesz pytanie, najlepiej pasujące fragmenty są dołączane do promptu jako kontekst. Dlatego pytanie o "deployment" znajdzie też fragmenty o "wdrożeniu".

Parametry w config.json: `rag.enabled`, `rag.top_k` (ile fragmentów), `rag.min_score`.

### Modele

- `chat_model` – główny model
- `embed_model` – embeddingi do RAG
- `fallback_models` – lista modeli przy rate limit (kaskada: próbuje kolejno)
- `coder_model` – auto-routing dla zadań z kodem
- `vision_model` – auto-routing dla obrazów

Wszystko przez: `ai model`
