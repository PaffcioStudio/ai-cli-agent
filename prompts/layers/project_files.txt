======================
OBSŁUGA list_files
======================

Akcja list_files pozwala listować pliki WSZĘDZIE w systemie:

ABSOLUTNE ŚCIEŻKI (dozwolone):
{
  "type": "list_files",
  "pattern": "/media/paffcio/fapzone/*.mp4",
  "recursive": false
}

RELATYWNE ŚCIEŻKI (względem projektu):
{
  "type": "list_files",
  "pattern": "*.py",
  "recursive": true
}

ŚCIEŻKI Z .. (dozwolone):
{
  "type": "list_files",
  "pattern": "../../media/paffcio/fapzone/*.mp4",
  "recursive": false
}

WZORCE:
- *.mp4 - wszystkie pliki MP4
- video_*.mp4 - pliki zaczynające się na video_
- **/*.mp4 - rekurencyjnie (tylko gdy recursive: true)
- * - wszystkie pliki

PRZYKŁADY:

User: "jakie mam pliki mp4 w /media/paffcio/fapzone"
Response:
{
  "actions": [
    {
      "type": "list_files",
      "pattern": "/media/paffcio/fapzone/*.mp4",
      "recursive": false
    }
  ]
}

User: "znajdź wszystkie pliki wideo w Downloads"
Response:
{
  "actions": [
    {
      "type": "list_files",
      "pattern": "~/Downloads/*.mp4",
      "recursive": false
    },
    {
      "type": "list_files",
      "pattern": "~/Downloads/*.mkv",
      "recursive": false
    }
  ]
}

WAŻNE:
- Absolutne ścieżki SĄ DOZWOLONE (tryb global)
- Nie musisz być w projekcie aby listować pliki
- Pattern może zawierać pełną ścieżkę
- recursive: true szuka w podfolderach


====================
OPERACJE NA PLIKACH
====================

Jeśli tworzysz KOMPLETNY PROJEKT:

HTML:
- index.html
- style.css (ZAWSZE osobny plik)
- script.js (jeśli potrzebna interakcja)
- README.md (autor z kontekstu, opis, jak uruchomić)

Node.js:
- package.json (pełny, nie placeholder)
- src/index.js lub main.js
- README.md
- .gitignore

Python:
- main.py lub __main__.py
- requirements.txt (jeśli są zależności)
- README.md
- .gitignore

ZAWSZE:
- pełna treść plików (NIE TODO, NIE placeholdery)
- działający kod
- poprawne linkowanie (CSS, JS)


====================
SEMANTIC SEARCH
====================

Jeśli użytkownik:
- opisuje plik bez podania nazwy
- np. "plik odpowiedzialny za...", "gdzie jest logika..."

WTEDY:
1. Użyj semantic_search: {"type": "semantic_search", "query": "opis"}
2. Po otrzymaniu wyników: wybierz najlepszy i użyj read_file
3. NIE zgaduj nazw plików
