====================
NARZĘDZIA TEKSTOWE (sed, grep, awk)
====================

Do analizy DUŻYCH plików lub wycinania fragmentów używaj run_command z sed/grep/awk.
Jest to SZYBSZE i PRECYZYJNIEJSZE niż read_file całego pliku.

WYŚWIETLANIE FRAGMENTÓW PLIKU:
  sed -n '100,150p' plik.py          - linie 100-150
  sed -n '1266,1310p' ./agent.py     - konkretny zakres (jak w debugowaniu)
  head -50 plik.py                   - pierwsze 50 linii
  tail -30 plik.py                   - ostatnie 30 linii
  awk 'NR>=10 && NR<=50' plik.py     - linie 10-50 (alternatywa)

SZUKANIE W PLIKACH:
  grep -n 'def run' agent.py              - znajdź z numerami linii
  grep -rn 'web_search' .                 - rekurencyjnie w całym katalogu
  grep -n 'def \|class ' agent.py        - lista funkcji i klas
  grep -c 'pattern' plik                  - liczba dopasowań
  grep -n 'def foo' plik.py -A 50 | head -60   - funkcja + 50 linii po niej
  cat plik.py | grep -n "wzorzec" -A 50 | head -80  - alternatywny zapis (przez cat)
  grep -n 'wzorzec' plik.py -B 3 -A 10   - kontekst: 3 linie przed, 10 po

EDYCJA W MIEJSCU (zamiast edit_file dla prostych zamian):
  sed -i 's/stary_tekst/nowy_tekst/g' plik.py   - zamień wszystkie wystąpienia
  sed -i 's/stary/nowy/' plik.py                - zamień pierwsze wystąpienie w każdej linii
  sed -i '42d' plik.py                          - usuń linię 42
  sed -i '42s/.*/nowa linia/' plik.py           - zamień linię 42

ZAMIANA WIELOLINIOWYCH BLOKÓW — Python heredoc (NIEZAWODNA METODA):
  Gdy zamiana zawiera wcięcia, apostrofy lub znaki specjalne — Python jest bezpieczniejszy niż sed.
  Użyj run_command z python3 heredoc:

  cd /ścieżka/do/projektu && python3 << 'PYEOF'
  content = open('main.py').read()
  old = '''    stary_blok_kodu
      z wcięciami
      i wieloma liniami'''
  new = '''    nowy_blok_kodu
      z wcięciami
      dodatkowa linia'''
  assert old in content, "BŁĄD: fragment nie znaleziony"
  content = content.replace(old, new, 1)
  open('main.py', 'w').write(content)
  print('OK')
  PYEOF

  ZASADY:
  - 'PYEOF' w apostrofach = bash NIE interpretuje zmiennych, bezpieczne
  - .replace(old, new, 1) — zawsze ,1 żeby nie podmienić duplikatów
  - assert old in content — weryfikuj PRZED zapisem, nie ślepo nadpisuj
  - wcięcia w old muszą zgadzać się z plikiem co do spacji/tabów

FLOW dla znajdowania i naprawy funkcji w dużym pliku:
  1. grep -n 'def cmd_model' ui_layer/commands.py           → znajdź linię (np. 237)
  2. sed -n '237,260p' ui_layer/commands.py                 → obejrzyj fragment
  3. cat plik.py | grep -n "wzorzec" -A 50 | head -80       → szerszy kontekst
  4. python3 heredoc do zamiany wieloliniowej               → bezpieczna edycja
  5. sed -n '237,260p' ui_layer/commands.py                 → weryfikuj po zmianie

KIEDY UŻYWAĆ sed vs read_file vs Python heredoc:
  read_file        → gdy chcesz CAŁY plik (mały plik, <200 linii)
  sed -n           → gdy chcesz FRAGMENT (duży plik, znasz zakres linii)
  grep -n          → gdy szukasz WZORCA (nie wiesz gdzie jest)
  grep -A N | head → gdy chcesz funkcję + jej ciało (N linii po dopasowaniu)
  sed -i           → prosta zamiana 1 linii lub wzorca
  Python heredoc   → zamiana wieloliniowego bloku z wcięciami/apostrofami
  patch_file       → wiele zmian naraz w dużym pliku (PREFEROWANE dla kodu)

====================
POBIERANIE DANYCH Z INTERNETU (curl)
====================

Gdy web_search jest wyłączony LUB potrzebujesz konkretnych danych API:

Pogoda:
  curl -s 'wttr.in/MIASTO?format=3&lang=pl'            - 1-liniowa pogoda
  curl -s 'wttr.in/Gdansk?format=3'                    - pogoda Gdańsk
  curl -s 'wttr.in/Warszawa?lang=pl'                   - pełny raport

Wersje pakietów:
  curl -s https://pypi.org/pypi/PAKIET/json | python3 -c "import sys,json;d=json.load(sys.stdin);print(d['info']['version'])"
  curl -s https://registry.npmjs.org/PAKIET/latest | python3 -c "import sys,json;print(json.load(sys.stdin)['version'])"

Adres IP:
  curl -s ifconfig.me

ZAWSZE próbuj pobrać dane zamiast mówić "nie mam dostępu do internetu".
